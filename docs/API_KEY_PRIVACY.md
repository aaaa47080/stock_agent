# API Key 儲存與隱私政策

## 概述

Pi CryptoMiner 允許用戶綁定自己的第三方 AI 服務 API Key（如 OpenAI、Google Gemini 等），以使用個人化的 AI 分析功能。本文檔說明我們如何處理和保護您的 API Key。

## 支援的 API Key 類型

- OpenAI API Key
- Google Gemini API Key  
- Anthropic API Key
- Groq API Key
- OpenRouter API Key

## 資料儲存方式

### 加密儲存

1. **傳輸加密**：所有 API Key 在傳輸過程中使用 HTTPS/TLS 1.3 加密
2. **靜態加密**：API Key 在資料庫中使用 AES-256 加密儲存
3. **加密金鑰管理**：
   - 加密金鑰透過 PBKDF2 演算法從伺服器端密鑰衍生
   - 金鑰不與用戶密碼或 token 綁定
   - 伺服器端密鑰儲存在環境變數中，不進入版本控制

### 儲存架構

```
用戶輸入 API Key
      ↓
HTTPS 加密傳輸
      ↓
後端 AES-256 加密
      ↓
加密後存入資料庫
```

### 資料保留

- API Key 會持續儲存直到您主動刪除
- 帳號刪除時，所有相關 API Key 會被永久刪除
- 我們不會在您刪除後保留任何 API Key 資料

## 我們如何使用您的 API Key

### 用途

您的 API Key 僅用於：
- 代表您調用對應的 AI 服務（如 OpenAI、Google）
- 執行您請求的 AI 分析功能

### 我們不會做的事

- ❌ 不會將您的 API Key 分享給第三方
- ❌ 不會使用您的 API Key 進行非您請求的操作
- ❌ 不會儲存您的 API Key 明文
- ❌ 不會將 API Key 用於任何其他目的

## 您的權利

### 查看與管理

- 隨時查看已綁定的 API Key 狀態（顯示遮蔽版本）
- 隨時更新或刪除您的 API Key
- 匯出您的 API Key 使用記錄

### 刪除權

- 您可以隨時在設定頁面刪除任何已綁定的 API Key
- 刪除後，相關加密資料會從資料庫永久移除

## 安全措施

### 技術措施

- AES-256-GCM 加密演算法
- PBKDF2 金鑰衍生（480,000 次迭代）
- HTTPS 強制加密傳輸
- 資料庫存取控制
- **定期金鑰輪換**：加密金鑰每 90 天自動輪換，降低金鑰洩露風險

### 營運措施

- API Key 不會出現在日誌中
- 前端只顯示遮蔽版本（如 `sk-****...****abc123`）
- 定期安全審計
- **金鑰輪換機制**：系統自動進行加密金鑰輪換，確保即使舊金鑰被破解，資料仍然安全

## 風險告知

### 用戶責任

1. **保管責任**：您應妥善保管自己的 API Key
2. **使用監控**：建議您定期檢查 AI 服務商的使用記錄
3. **配額管理**：調用次數受您 API Key 的配額限制

### 潛在風險

- 若您的帳號被盜，攻擊者可能使用您的 API Key
- 建議設定 AI 服務商的使用限額和預算警報

### 建議

- 定期檢查您的 API Key 使用情況
- 設定 API Key 的使用限額
- 如發現異常使用，立即刪除並重新生成 Key
