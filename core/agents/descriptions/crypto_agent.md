# Crypto Agent

## When to Use
當用戶的查詢涉及：
- 數位貨幣或加密資產
- 區塊鏈相關代幣
- 去中心化金融 (DeFi) 相關資訊
- 加密貨幣交易所數據
- **加密貨幣市場的恐慌貪婪指數**（當對話上下文涉及加密貨幣時）

## Capabilities
- 即時價格查詢
- 技術指標分析 (RSI, MACD, 均線等)
- 合約數據 (資金費率、持倉量)
- 鏈上數據 (Gas 費用、鯨魚交易、交易所流向)
- **加密貨幣恐慌貪婪指數** (Crypto Fear & Greed Index) - 專門針對加密市場
- 熱門幣種排行
- ⚠️ **新聞資訊** - 使用 google_news 或 aggregate_news 工具
- 代幣解鎖日程
- TVL 數據

## ⚠️ 與 Economic Agent 的邊界
- **加密貨幣恐慌貪婪指數** → Crypto Agent（本 Agent）
- **傳統金融 VIX 恐慌指數** → Economic Agent
- 當用戶在討論加密貨幣（如 BTC、ETH）時提到「恐慌指數」，應理解為加密貨幣恐慌貪婪指數

## ⚠️ 綜合分析任務規劃

當用戶問「值得投資嗎」「可以買嗎」「完整分析」「整體評估」時，需要創建多個任務：

1. **價格查詢** - 任務描述：「查詢 [標的] 的即時價格」
2. **技術分析** - 任務描述：「對 [標的] 進行技術指標分析」
3. **市場情緒** - 任務描述：「獲取 [標的] 的市場情緒與恐慌貪婪指數」
4. **新聞資訊** - 任務描述：「查詢 [標的] 的最新市場新聞」

⚠️ **重要**：綜合分析必須包含以上 4 個任務，缺一不可！

## ⚠️ 新聞查詢任務規劃

當用戶問題包含以下關鍵字時，任務描述必須明確提到「新聞」：
- 「新聞」「消息」「動態」「資訊」「事件」
- 「最新」「最近」「近期」+ 市場相關
- 「發生了什麼」「有什麼消息」

範例：
- 用戶問「BTC 最新新聞」→ 任務描述應為「查詢 BTC 的最新新聞」
- 用戶問「比特幣有什麼消息」→ 任務描述應為「獲取比特幣的市場消息與動態」

## Market Indicators
用戶提到任何數位資產、區塊鏈代幣、加密貨幣相關術語時，應路由至此 agent。
